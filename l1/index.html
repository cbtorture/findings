L1: List initializer element evaluation ordering


Description

The order in which the elements in a list initializer are evaluated is unspecified. Here we will be looking at a piece of code which results in different outcomes depending on the order in which the elements in the list initializer are evaluated. Seeing that there is implicit order in an array we will be able to see if the evaluation order differs by the placement of returned results in the array:

int result[2] = {counter(), counter()};

L1 is very basic, we are using only two elements in the initializer for result[2] and also the same funciton is being used, counter(), as the element initializer.

Implicit version

The implicit version of the program will be the following:

#include<assert.h>
#include<stdio.h>

/* Global data */
int count = 0;

/* Counting function */
int counter()
{
    count++;
    return count;
}

int main()
{
    /**
    * Calculate the array
    *
    * Ex1: {1, 2}
    * Ex2: {2, 1}
    */
    int result[2] = {counter(), counter()};

    /**
    * assert(result[0] == 1);
    * assert(result[1] == 2);
    */

    /**
    * assert(result[0] == 2);
    * assert(result[1] == 1);
    */
}

We will be testing a two versions of this same program to account for the two possible outcomes (and also what they mean semantically):

  1. result[0] == 1 && result[1] == 2
    • This implies left-to-right evaluation
  2. result[0] == 2 && result[1] == 1
    • This implies right-to-left evaluation

Analysis

Every compiler always does left-to-right evaluation.



Explicit versions

To improve CSeq per-compiler behaviour we will need explicit versions that provide the given behaviour, in this case there will be only two explicit versions.

right-to-left execution ordering


#include<assert.h>
#include<stdio.h>

/* Global data */
int count = 0;

/* Counting function */
int counter()
{
    count++;
    return count;
}

int main()
{
    int res1_val = counter();
    int res2_val = counter();
    int result[2] = {res2_val, res1_val};

    assert(result[0] == 2);
    assert(result[1] == 1);
}
